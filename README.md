# 📋 Task Management (SQLite + Python + Poetry)

Простий проєкт для роботи з базою даних SQLite: створення таблиць, генерація тестових даних та виконання SQL-запитів через Python.

---

## 📂 Структура проєкту

```
.
├── tasks.sql        # SQL-скрипт створення таблиць
├── create_db.py     # створення бази даних
├── seed.py          # заповнення БД фейковими даними
├── select.py        # запити до бази даних
├── pyproject.toml   # конфігурація Poetry
├── tasks.db         # (генерується автоматично)
└── README.md
```

---

## 🗄️ Структура бази даних

### Таблиця `users`

* `id` — PRIMARY KEY
* `fullname` — ім’я користувача
* `email` — унікальний email

### Таблиця `status`

* `id` — PRIMARY KEY
* `name` — статус задачі:

  * `new`
  * `in progress`
  * `completed`

### Таблиця `tasks`

* `id` — PRIMARY KEY
* `title` — назва задачі
* `description` — опис
* `status_id` — FOREIGN KEY → status
* `user_id` — FOREIGN KEY → users

## 🚀 Як запустити проєкт

### 1. Встановити Poetry

```
pip install poetry

or

poetry init
```

---

### 2. Встановити залежності

```
poetry install --no-root
```

---

### 3. Активувати середовище

```
poetry shell
```

---

### 4. Створити базу даних

```
python create_db.py
```

✔️ Створить `tasks.db` і таблиці.

---

### 5. Заповнити даними

```
python seed.py
```

✔️ Додає:

* користувачів
* задачі
* статуси

---

### 6. Виконати запити

```
python select.py
```

✔️ Виконує 14 запитів і виводить результати.

---

## 🎲 Генерація даних (Faker)

Проєкт використовує бібліотеку **Faker** для створення тестових даних.

Генерується:

* випадкові імена (`fullname`)
* email адреси
* назви задач
* описи задач

Також:

* задачі випадково прив’язуються до користувачів
* статуси призначаються випадково

---

## 🔍 Реалізовані запити

1. Завдання конкретного користувача
2. Завдання зі статусом `new`
3. Оновлення статусу задачі
4. Користувачі без задач
5. Додавання нової задачі
6. Незавершені задачі
7. Видалення задачі
8. Пошук користувачів по email
9. Оновлення імені користувача
10. Кількість задач по статусах
11. Задачі за доменом email
12. Задачі без опису
13. Задачі `in progress` з користувачами
14. Кількість задач у кожного користувача
